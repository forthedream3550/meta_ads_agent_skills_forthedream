# Public and private boundary

## Safe to publish

- generic skills and workflows;
- schemas;
- code without credentials;
- fictional examples;
- general safety rules;
- public documentation links;
- provider interfaces;
- test data.

## Keep private

- customer names and account identifiers;
- access tokens and app secrets;
- exact internal benchmarks;
- customer margins and lead values;
- CRM exports;
- proprietary sector conclusions;
- internal approval identities;
- production audit logs;
- private creative performance history.

## Configuration model

Keep the public repository opinionated about method and conservative about execution. Keep account strategy and commercial thresholds in private deployment configuration.
